What is a 'path' in causal mapping?
y A sequence of causal links connecting one factor to another
A single factor that has many consequences
A list of all the sources that mention a factor
A factor that appears at the start of the map

What does the path tracing filter do?
y It keeps only links that lie on a pathway between the factors you choose
It automatically adds new links between factors that seem related
It deletes every factor that is not an outcome
It turns the map into a table of sources
hint Path tracing is link-based: it keeps existing links, it does not invent new ones.

In the path tracing filter, what happens if you set a 'From' factor but leave the 'To' factor blank?
y You see all downstream pathways leading out of the From factor
You see nothing, because both fields are required
You see only direct links, never indirect ones
The filter traces paths backwards into the From factor

Why is it useful to keep the maximum number of steps (K) fairly small?
y Chains longer than about four steps are uncommon and hard to interpret
Setting K to a higher value forces the app to recalculate the entire dataset from scratch each time you adjust the filter
A large K deletes links from the map
hint When analysing interviews, people rarely report causal chains longer than about four steps.

A map shows A --> B and B --> C. Can you conclude that at least one respondent claimed A leads to C?
No, you can never combine links in causal mapping under any circumstances, even when reviewing a single transcript
y No, the two links may come from different sources
Yes, links are always transitive
hint This is the transitivity trap: the two links may come from different sources.

What is the difference between standard path tracing and source tracing?
y Source tracing keeps only pathways found within a single source; standard tracing can stitch a path across several sources
Source tracing reweights each link by the number of sources that mention it, then removes any link below a configurable threshold before displaying results
There is no difference, they are two names for the same thing
Standard tracing only ever works on one source at a time

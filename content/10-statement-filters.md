What is the primary function of statement filters in Causal Map?
y To refine the data set by including or excluding statements based on criteria.
To change the colour and display style of statements for visual differentiation between sources.
To automatically translate statements into different languages.
To increase the font size of statements.

How can statement filters enhance the analysis process in Causal Map?
y By allowing users to focus on relevant data and exclude irrelevant information.
By enabling automatic coding of all statements without any user intervention or review.
By summarising statements into a single report.
By converting qualitative data into quantitative data automatically.

Can statement filters in Causal Map be customised for specific research needs?
y Yes, users can create custom filters based on various criteria.
No, statement filters are fixed and cannot be adjusted.
Yes, but only by contacting Causal Map support.
No, filters are applied automatically and cannot be altered.

What is an example of a criterion that could be used in a statement filter?
y The presence of specific keywords or phrases within the statements.
y Showing only data from a specific source or sources.
y Showing only data from sources with particular characteristics, e.g. all women.
y Showing only data related to a specific question or questions.
y Showing only coded statements.
y Showing only uncoded statements.
The length of the statements, measured in number of words, used to exclude very short or very long responses from the map.

How do statement filters interact with the coding process in Causal Map?
y Filters can be applied during coding if desired.
y Filters can be applied after coding to analyse subsets of the data.
Filters automatically code data without user intervention.

When you export or read a pivot/table, aligning field names matters because:
y Misreading 'source counts' versus 'quote counts' (etc.) silently changes interpretations
Treat similarly worded column titles as interchangeable — respondent counts, excerpt counts, and filtered link rows blur when headers almost match.
Rounded numbers on slides are harmless to paraphrase — you skip checking weights, denominators, or hidden small-cell rules behind the pivot.
Blame collaborator Excel tweaks — column mismatch must be manual renames — before rechecking the app's documented field meanings.

A beginner-proof habit is:
y Skim garden/app docs for glossary columns before presenting numbers externally
Rely on generic Excel instincts; give header text a skim only when numbers already confirm what you hoped.
Tidy tables feel self-explanatory — weighting notes and denominators get dropped when you're rushing.
Drop headers and axes on PDF decks to gain space — readers won't need to know which filter produced the table.

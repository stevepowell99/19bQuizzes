When you export or read a pivot/table, aligning field names matters because:
y Misreading 'source counts' versus 'quote counts' (etc.) silently changes interpretations
Treating every similarly worded column header as interchangeable—even when one tab counts distinct respondents, another counts coded quotations, and a third counts raw link rows under different exclusions
Assuming decimal precision is decorative so rounded cells can be paraphrased freely in slide bullets without reconciling definitions of weighting, percentages, or suppressed small cells
Inferring that collaborator bookmarks or ad hoc renames in personal spreadsheets have silently overwritten canonical field names in shared exports, rather than misreading the app's documented glossary

A beginner-proof habit is:
y Skim garden/app docs for glossary columns before presenting numbers externally
Presetting mental models from generic spreadsheet software conventions and only glancing at headers when figures already match prior expectations
Leaving units, weighting rules, and denominators implicit in external briefings because tidy tables look self-explanatory without double-checking methodology notes
Removing axis labels or table headers in PDF decks to save space even when partners later cannot tell which filter state produced the displayed counts
